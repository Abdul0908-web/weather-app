# Security Policy

## Reporting a Vulnerability

Although this is a personal portfolio project, security issues are taken seriously.

If you discover a vulnerability, please:
1. Open a GitHub issue.
2. Clearly describe the issue.
3. Provide steps to reproduce it.

I will review and address valid reports in a timely manner.

## Notes

- API keys are not stored in this repository.
- Sensitive data should not be committed to version control.
